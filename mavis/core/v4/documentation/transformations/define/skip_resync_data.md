
## What is Skip Resync Data?

This will push the transformation to production but not trigger a resync (no deleting) and will continue updating the data moving forward.

------

# FAQ


**When should I use it?**

When you optimize a query!

Sometimes you want to optimize a SQL query to be more performant but the actual output doesn't change, so there is no need to resync all the data.  You can just use this new SQL moving forward.
